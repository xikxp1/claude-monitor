# Architecture

Codex Monitor is an Electrobun desktop app with a SvelteKit 5 static frontend and a Bun main process. The active desktop runtime is Electrobun; the old Tauri/Rust code is no longer on the app path.

## Frontend

- Located in `src/`.
- Uses Svelte 5 runes, TypeScript, Tailwind, and daisyUI.
- Built as a static SPA into `web-build/` and copied into the Electrobun bundle under `views/mainview/`.
- Talks to the Bun backend through typed Electrobun RPC in `src/lib/electrobunClient.ts`.
- Shared public data contracts live in `src/shared/types.ts`.

## Bun Backend

- Located in `src/bun/`.
- `index.ts` creates the hidden borderless dashboard window, tray icon, RPC handlers, and auto-refresh loop.
- Provider fetchers use `fetch`:
  - Claude: `GET https://claude.ai/api/organizations/{org_id}/usage`
  - Codex: `GET https://chatgpt.com/backend-api/wham/usage`
  - Ollama: HTML scraping from `https://ollama.com/settings`
- Settings are JSON at `Utils.paths.appData/settings.json`.
- Sensitive Claude and Ollama credentials use `@napi-rs/keyring` with service `dev.xikxp1.claude-monitor`, preserving the previous keychain identity without passing session tokens through shell command arguments.
- Usage history uses `bun:sqlite` at `Utils.paths.appData/usage_history.db` and preserves the normalized `usage_history_v2` schema.
- Notifications use `Utils.showNotification`; fired threshold/time-remaining state is persisted to `notification_state` in settings after refreshes so dedupe survives restarts.
- Updates use Electrobun `Updater`.
- Autostart is implemented with platform adapters:
  - macOS LaunchAgent
  - Windows `Run` registry key
  - Linux `.desktop` autostart file

## Tray And Window Lifecycle

- Electrobun owns the system tray and hidden utility window.
- Non-macOS positions the borderless window near `Tray.getBounds()` and hides it by moving it offscreen.
- macOS routes tray clicks through `src/bun/native/popover.ts` into `native/macos/ClaudeMonitorPopover.m`, which anchors a real `NSPopover` to Electrobun's native `Tray.ptr` and hosts the dashboard window content inside it.
- `runtime.exitOnLastWindowClosed = false` keeps the tray app alive.

## Build And Release

```bash
bun run check
bun run test
bun run build
```

`bun run build` runs `vite build`, then `electrobun build --env=stable`. Release artifacts and `update.json` are generated by Electrobun from `electrobun.config.ts`.
